# 开发环境搭建

> 目标芯片：**STM32F103C8T6**。工具链：**CMake + arm-none-eabi-gcc + Ninja**，
> 编辑器 **VSCode**，烧录调试 **ST-Link + OpenOCD**。

---

## 0. 总览

需要安装的工具及其用途：

| 工具 | 用途 | 版本要求 |
| --- | --- | --- |
| **Git** | 拉取代码、获取 HAL | 较新版本即可 |
| **CMake** | 构建系统 | **≥ 3.21**（使用 CMakePresets） |
| **Ninja** | 构建执行器（比 make 快） | 较新版本即可 |
| **arm-none-eabi-gcc** | ARM 交叉编译器（编译固件） | GNU Arm Embedded Toolchain |
| **OpenOCD** | 烧录 / 调试 STM32 | ≥ 0.11 |
| **VSCode** | 代码编辑器 | 最新版 |
| host 编译器（gcc/clang/MSVC） | 在 PC 上编译 algorithms 跑单元测试 | 建议安装 |

**HAL 库（STM32CubeF1）不纳入版本库**，由 `scripts/fetch_hal.sh` 按需拉取（仅 sparse-clone
`Drivers` 目录，约数十 MB）到 `core/STM32CubeF1/`。`bootstrap` 脚本会自动完成；仅跑 host 测试则无需它。

### 最快路径（工具装好后）

```bash
git clone <仓库地址> health-terminal
cd health-terminal
./scripts/bootstrap.sh      # Linux / macOS
# Windows PowerShell：.\scripts\bootstrap.ps1
```

`bootstrap` 会检查工具是否齐全、获取 HAL 并完成首次配置。它能跑通即表示环境就绪。

## 1. Windows 安装（多数同学使用）

逐项安装，每步装完执行**验证命令**确认，看到版本号才算成功。

> 安装涉及 PATH 的工具后需**新开终端**（关闭旧窗口重开），环境变量方可生效。

### 1.1 Git

<https://git-scm.com/download/win> 下载安装，默认下一步即可。验证：`git --version`

### 1.2 CMake

<https://cmake.org/download/> 下载 Windows x64 `.msi`。安装时**勾选
"Add CMake to the system PATH for all users"**。验证：`cmake --version`（需 ≥ 3.21）

### 1.3 Ninja

三选一：

- scoop：`scoop install ninja`
- choco：`choco install ninja`
- 手动：<https://github.com/ninja-build/ninja/releases> 下载 `ninja-win.zip`，解压 `ninja.exe`
  至固定目录（如 `C:\tools\ninja\`），将该目录加入 PATH。

验证：`ninja --version`

### 1.4 Arm GNU Toolchain

<https://developer.arm.com/downloads/-/arm-gnu-toolchain-downloads> 下载 **arm-none-eabi**
的 Windows `.exe`。安装最后一步**勾选 "Add path to environment variable"**。
验证：`arm-none-eabi-gcc --version`（应显示 13.x 等版本号）

### 1.5 OpenOCD

Windows 无官方安装器，二选一，解压后将 `bin` 目录加入 PATH：

- xpack-openocd（推荐）：<https://github.com/xpack-dev-tools/openocd-xpack/releases>
- openocd-mingw：<https://gnutoolchains.com/arm-eabi/openocd/>

验证：`openocd --version`

### 1.6 host 编译器（跑单元测试用）

用于在 PC 上编译 `algorithms` 并跑单元测试（**无需开发板**），二选一：

- MSVC Build Tools（推荐）：<https://visualstudio.microsoft.com/downloads/> 下载
  "Build Tools for Visual Studio"，勾选"使用 C++ 的桌面开发"。
- MinGW-w64 gcc：`scoop install gcc` 或 `choco install mingw`。

### 1.7 ST-Link 驱动

插入 ST-Link 后需正确驱动，二选一：

- ST 官方驱动：搜索 "STSW-LINK009" 下载安装。
- Zadig：<https://zadig.akeo.ie/>，选中 ST-Link 安装 WinUSB 驱动（配合 OpenOCD 更省心）。

### 1.8 替代方案：scoop 一键装齐

安装 [scoop](https://scoop.sh/) 后：

```bash
scoop bucket add main
scoop install git cmake ninja gcc-arm-none-eabi openocd
```

（host 编译器与 ST-Link 驱动仍需按上文单独安装。）

## 2. macOS 安装

用 [Homebrew](https://brew.sh/)：

```bash
brew install cmake ninja open-ocd
```

## 4. VSCode 配置

### 4.1 安装扩展

- **C/C++**（`ms-vscode.cpptools`）
- **CMake Tools**（`ms-vscode.cmake-tools`）
- **Cortex-Debug**（`marus25.cortex-debug`）— 用于烧录调试

### 4.2 选择 Preset 构建

`Ctrl+Shift+P` → `CMake: Select Configure Preset`，选择：

- `host` — 在 PC 上编译并跑单元测试
- `stm32` — ARM 交叉编译生成固件

选好后点状态栏 **Build** 即可编译。

### 4.3 调试配置（可选）

在根目录创建 `.vscode/launch.json`，配置 Cortex-Debug：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug STM32 (OpenOCD)",
      "type": "cortex-debug",
      "request": "launch",
      "servertype": "openocd",
      "cwd": "${workspaceFolder}",
      "executable": "build/stm32/firmware.elf",
      "device": "STM32F103C8",
      "interface": "swd",
      "configFiles": ["interface/stlink.cfg", "target/stm32f1x.cfg"],
      "runToEntryPoint": "main"
    }
  ]
}
```

> 需先完成第 7 节交叉编译、`build/stm32/firmware.elf` 存在，方可调试。

## 5. 拉取代码与获取 HAL

```bash
git clone <仓库地址> health-terminal
cd health-terminal
```

编译固件前须获取一次 HAL（仅跑 host 测试可跳过）。`core/STM32CubeF1` 不存在时执行：

```bash
bash scripts/fetch_hal.sh
```

> CubeF1 主仓将 HAL Driver 与 CMSIS Device 做成嵌套子模块，整仓（含 DSP/例程）超过 1GB。
> 该脚本仅浅克隆并 sparse 拉取所需部分（约 27MB），显著加快获取。网络较慢时请耐心等待。

也可直接运行 `./scripts/bootstrap.sh`，其内部会自动获取 HAL。

## 6. 编译 host 单元测试（无需开发板）

在 PC 上编译并运行 `algorithms` 的单元测试，是验证环境是否配好的最简方式：

```bash
cmake --preset host
cmake --build --preset host
ctest --preset host --output-on-failure
```

预期 `test_filters` / `test_codec` / `test_algorithms` 全部 **Passed**：

```text
100% tests passed, 0 tests failed out of 3
```

## 7. 交叉编译固件

```bash
cmake --preset stm32
cmake --build --preset stm32
```

产物在 `build/stm32/`：`firmware.elf`（带调试信息，供 OpenOCD/GDB）、`firmware.bin`（纯二进制）、
`firmware.hex`（Intel HEX）。编译结束会打印 Flash / RAM 占用。

### MODULE_SET：只编译自己负责的模块

`stm32` preset 提供缓存变量 `MODULE_SET`，用于选择编译的模块子集，避免每人都编译全部代码。
可选 `yuhao` / `chafanting` / `liuyanming` / `full`：

```bash
cmake --preset stm32 -DMODULE_SET=yuhao
cmake --build --preset stm32
```

`full` 为全部模块（集成联调时用）。不指定时使用 preset 默认值。

## 8. 烧录与调试（ST-Link + OpenOCD）

### 8.1 接线

**ST-Link ↔ 开发板（SWD 4 线）：**

| ST-Link | STM32F103 |
| --- | --- |
| SWDIO | PA13 |
| SWCLK | PA14 |
| GND | GND |
| 3.3V | 3V3 |

**USB-TTL 串口 ↔ 开发板（查看日志）：**

| USB-TTL | STM32F103 |
| --- | --- |
| RX | PA9 (USART1 TX) |
| GND | GND |

> **TX 与 RX 交叉连接**：开发板 TX(PA9) 接串口模块的 RX。

### 8.2 一条命令烧录

```bash
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg -c "program build/stm32/firmware.elf verify reset exit"
```

出现 `Verified OK` 与 `** Resetting Target **` 即烧录成功。

### 8.3 命令行调试（可选）

一个终端启动 OpenOCD（监听 3333 端口的 gdb server）：

```bash
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg
```

另一终端连接调试：

```bash
arm-none-eabi-gdb build/stm32/firmware.elf
(gdb) target extended-remote localhost:3333
(gdb) load
(gdb) monitor reset halt
(gdb) continue
```

> 使用 VSCode 时按 4.3 节配置好后直接 F5，无需手敲 gdb。

### 8.4 首版固件的预期现象

- **PC13 板载 LED 闪烁**，用于判断程序是否运行。
- **USART1 打印版本号**，串口参数 **115200 8N1**（115200 波特、8 数据位、无校验、1 停止位）。

用串口助手 / PuTTY / screen 查看，例如：

```bash
screen /dev/ttyUSB0 115200      # 退出：Ctrl+A 后按 K，再按 Y
```

Windows 用 PuTTY 选对应 COM 口、波特率 115200。

## 9. 常见问题

**`arm-none-eabi-gcc: command not found`**
编译器未进入 PATH。检查安装时是否勾选 "Add path to environment variable"，或手动将 toolchain
的 `bin` 加入 PATH，然后**新开终端**重试。

**`ninja: command not found` / CMake 报 "Could not find Ninja"**
Ninja 未安装或未进入 PATH。按 1.3 重装，将 `ninja.exe` 目录加入 PATH，新开终端。

**配置 stm32 时报找不到 HAL / STM32CubeF1**
多为 HAL 尚未获取（`core/STM32CubeF1` 不存在或为空）。执行 `bash scripts/fetch_hal.sh` 后
重新 `cmake --preset stm32`。

**OpenOCD 找不到 ST-Link（`open failed`）**
- Windows：驱动未装对，用 ST 官方驱动或 Zadig 装 WinUSB（见 1.7）。
- Linux：多为权限问题，创建 `/etc/udev/rules.d/49-stlink.rules`：

  ```text
  SUBSYSTEM=="usb", ATTR{idVendor}=="0483", ATTR{idProduct}=="3748", MODE="0666"
  SUBSYSTEM=="usb", ATTR{idVendor}=="0483", ATTR{idProduct}=="374b", MODE="0666"
  ```

  然后 `sudo udevadm control --reload-rules && sudo udevadm trigger`，重新插拔 ST-Link。

**串口无输出**
逐项排查：① 波特率是否 115200 8N1；② TX/RX 是否接反（PA9=TX 接串口模块 RX）；③ 是否共地；
④ 固件是否真正烧入（PC13 LED 是否闪烁）；⑤ 串口设备是否选对（Windows 看设备管理器 COM 号，
Linux 一般为 `/dev/ttyUSB0`）。
