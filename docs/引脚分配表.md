# 整机引脚分配表（STM32F103C8T6 / LQFP48）

> 本表为**第一周冻结的契约**，一旦定稿三人接线一律照此执行。任何改动必须同步
> `core/include/board.h` 并全员周知。与 `board.h` 中的宏定义严格一一对应。

## 0. 芯片资源与总原则
- STM32F103C8T6：Cortex-M3 @72MHz，Flash 64KB，RAM 20KB，LQFP48 封装。
- 严格保留 **PA13/PA14 = SWD**（SWDIO/SWCLK），用于烧录调试，任何模块不得占用。
- **PD0/PD1 = OSC_IN/OSC_OUT**，接 8MHz HSE 晶振，不作 GPIO。
- **PC14/PC15**（OSC32）、**PA11/PA12**（USB DM/DP）保留为空闲备用，尽量不占用。
- 5V 容忍（FT）校验：HC-SR04 的 Echo 与 HX711 的 DOUT 若为 5V 电平，必须落在 FT 引脚。
  F103 上**非 FT 引脚**主要是带 ADC 的模拟脚：PA0–PA7、PB0、PB1；其余数字脚基本为 FT。

## 2. 逐条分配理由与校验

### 2.1 SWD 与时钟（不可动）
PA13/PA14 固定 SWD；PD0/PD1 接 8MHz 晶振（PLL×9→72MHz，见 `core/src/system_clock.c`）。
固件在 `HAL_MspInit` 中执行 `__HAL_AFIO_REMAP_SWJ_NOJTAG()`：**只关 JTAG、保留 SWD**，从而释放
PA15/PB3/PB4 作普通 IO，同时不影响烧录。

### 2.2 I2C1 共享总线（PB6/PB7）
三个 I2C 从机地址互不冲突（0x3C/0x68/0x57），单条总线即可挂载，符合"I2C1 共享总线"约束。
用 I2C1 默认脚 PB6/PB7（不重映射），把 PB8/PB9 让给 HC-SR04。二者均为 FT，接 3.3V 上拉电阻
（OLED/传感器模块多自带上拉，整机只保留一组 4.7kΩ 上拉，避免并联过强）。总线速率取 100kHz
标准模式，兼容长杜邦线与多从机（见 `bsp/i2c_bus.c`）。

### 2.3 两路 USART（复用核对）
- **USART1（PA9/PA10）** 作 printf 调试，115200 8N1。PA9/PA10 为 FT。
- **USART2（PA2/PA3）** 接 BLE（JDY-23/HM-10），9600 波特（HM-10 默认）。
两路 USART 引脚彼此独立、且不与 I2C1(PB6/7)、HX711、键盘冲突。满足"另保留一路 USART 做 printf"。

### 2.4 两路 HX711（引脚独立）
HX711 为 2 线软件时序（SCK 输出 + DOUT 输入），两路引脚完全独立：
- #1 体重：SCK=PA1、DOUT=PB11；#2 握力：SCK=PA0、DOUT=PB10。
- **FT 校验**：DOUT 是 HX711 输出，若 HX711 供 5V 则 DOUT 为 5V，故 DOUT 一律放 FT 脚（PB10/PB11 均 FT）。
  SCK 是 MCU→HX711 的输出（3.3V 驱动，HX711 可识别），非 FT 的 PA0/PA1 用作输出不涉及 5V 容忍问题。
- 建议 HX711 供 3.3V 可进一步降低干扰；两路模块地线与体重/握力传感器分别就近汇流。

### 2.5 HC-SR04（FT 强约束）
- Trig=PB9（输出，10µs 触发脉冲）。
- **Echo=PB8**：HC-SR04 的 Echo 为 **5V 输出**，PB8 为 **FT 引脚**，可直连，无需分压。
  （若换用非 FT 脚，必须 1kΩ+2kΩ 分压到 3.3V；本表已避免此情况。）
- PB8 复用 **TIM4_CH3 输入捕获**测回波脉宽，换算距离；配合 DS18B20 温度修正声速。
  中断走 `TIM4_IRQHandler`（见 `core/src/stm32f1xx_it.c`）。
- 稳妥起见可在 Echo 串接 330Ω 限流电阻保护引脚。

### 2.6 DS18B20（单总线）
DQ=PB5，开漏 + 4.7kΩ 上拉到 3.3V。PB5 为 FT，DS18B20 供 3.3V 时 DQ 为 3.3V。1 个 GPIO 完成单总线时序。

### 2.7 4×4 矩阵键盘（8 GPIO）
- 行 ROW0..3 = PB12..PB15（推挽输出，逐行拉低扫描）。
- 列 COL0..3 = PA4..PA7（上拉输入，读回哪列为低即按下）。
- 采用**轮询扫描**，不占用 EXTI，避免与反应按键的 EXTI 线冲突。
- PA4..PA7 非 FT，但键盘为 3.3V 逻辑（列脚仅读回被行拉低的 3.3V/GND），不涉及 5V，安全。

### 2.8 反应时间测试（独立 LED + EXTI 按键）
- 独立 LED = PB1（输出，区别于 PC13 状态灯，满足"独立 LED"要求）。
- 独立按键 = PB0，上拉输入并接 **EXTI0**（按下产生外部中断，精确记时刻），满足"须支持 EXTI"。
- **EXTI 线冲突检查**：本工程仅用到 EXTI line0（PB0）。EXTI line0 由 PA0/PB0/PC0 共享，
  而 PA0 已用作 HX711#2 的 **SCK 输出**（不接 EXTI）、PC0 未用，故 line0 唯一归属 PB0，**无冲突**。
  键盘走轮询、I2C/HX711 无 EXTI，全局不存在第二条 EXTI 线争用。

### 2.9 片内 Flash 模拟 EEPROM（无引脚，链接脚本隔离）
历史记录不占引脚：在 `core/ldscripts/STM32F103C8Tx_FLASH.ld` 中把 64KB Flash 的
**最后 4 页（4KB，0x0800F000–0x0800FFFF）** 划为 `EEPROM` 段，代码区限定为前 60KB，
二者物理隔离。`storage` 模块经 `bsp/flash_eeprom` 只在该区读写（见对应文件）。
