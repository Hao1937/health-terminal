# 三人第一周行动清单（Day 1 ~ Day 7）

> 目标：一周内**人人环境跑通、首版固件上板、各自认领模块并把第一个桩替换成真实现**。
> 每天结束前 `git commit` 一次；周日为合并日（见 [协作流程](协作流程.md)）。

## 共同前置（Day 1，三人都做）
1. 按 [开发环境搭建.md](开发环境搭建.md) 装齐工具链：Git、CMake、Ninja、arm-none-eabi-gcc、OpenOCD、VSCode（+host 编译器）。
2. `git clone` 克隆仓库，或克隆后 `bash scripts/fetch_hal.sh`。
3. 跑通 host 自检：`cmake --preset host && cmake --build --preset host && ctest --preset host` —— 应看到
   `test_filters / test_codec / test_algorithms` 全部 Passed。
4. **烧 hello 固件**：`cmake --preset stm32 && cmake --build --preset stm32`，用 OpenOCD 烧录，
   确认 **PC13 LED 闪烁** + USART1(115200) 打印 `health-terminal v0.1` 与自检信息。
5. 通读 [接口文档.md](接口文档.md) 与 [引脚分配表.md](引脚分配表.md)，理解 `hs_status_t`、
   `hs_sensor_t` 接口与 `measurement_record_t`/帧格式（**本周冻结，不得擅改**）。
6. 从 `dev` 切出各自分支：`git checkout dev && git checkout -b feature/<姓名>-<模块>`。

## 王宇浩（组长）—— bsp / 滤波 / MAX30102 / 握力 / 评分 / 集成
- **Day 1**：完成共同前置；确认 `main` 保护、建好 `dev`，把首版可编译工程推上远端。
- **Day 2**：核对 `bsp/`（i2c_bus/uart/gpio/delay/flash_eeprom）在真板上可用：I2C 扫描能探到
  OLED/MPU6050/MAX30102 三个地址（看 `app_init` 打印的 `I2C probe`）；`delay_us` 用逻辑分析仪校准。
- **Day 3**：`algorithms/filters` 已含单元测试；在 PC 上再补 1~2 组真实录制数据回归，确认滤波库可交付他人使用。
- **Day 4~5**：接 MAX30102（I2C 0x57），把 `modules/max30102.c` 的 `MODULE_ENABLED_MAX30102`
  分支从桩替换为真实现：读 FIFO 得 RED/IR 缓冲 → 调 `spo2_hr_compute` → `out->primary=心率, secondary=SpO2x10`。
  用 `cmake --preset stm32 -DMODULE_SET=yuhao` 单编自己模块烧板，**实测静息心率并记录**。
- **Day 6**：接 HX711#2（握力，PA0/PB10），实现 `hx711_grip_measure` 时序 + 标定（kg×10），实测握力值。
- **Day 7**：整理 `health_score` 权重；汇总本周三人进度，主持第一次合并日，把稳定点合入 `dev`。

## 查樊听 —— 体重(HX711#1) / 身高(HC-SR04) / 温度(DS18B20)
- **Day 1**：完成共同前置，烧 hello 固件确认板子 OK；切分支 `feature/chafanting-hx711-weight`。
- **Day 2**：接 HX711#1（体重，PA1/PB11）。先用示波器/逻辑分析仪看 SCK/DOUT 波形，跑通 24bit 读数。
- **Day 3**：实现 `modules/hx711_weight.c` 真实现（`MODULE_ENABLED_HX711_WEIGHT` 分支）：读原始值→
  减皮重→乘标定系数得克数写 `out->primary`；不稳定时返回 `HS_UNSTABLE`。`-DMODULE_SET=chafanting` 单编烧板。
- **Day 4**：**标定体重**：用标准砝码（如 5/10/20kg）做两点标定，记录标定系数与误差，作为中期实测数据。
- **Day 5**：接 HC-SR04（Trig=PB9/Echo=PB8），用 TIM4_CH3 输入捕获测回波，换算距离，实现 `hcsr04_measure`。
- **Day 6**：接 DS18B20（PB5），实现 `ds18b20_measure` 单总线时序读温度；用温度**补偿声速**修正 HC-SR04 身高。
- **Day 7**：整理身高/体重/温度单体实测数据；提交并参加合并日。
